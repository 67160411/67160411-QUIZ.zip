# ข้อที่ 1 สถาปัตยกรรมเว็บและ HTTP

### อธิบายกระบวนการสื่อสาร HTTP/HTTPS เมื่อผู้ใช้ป้อน URL ในเบราว์เซอร์ จากการส่งคำขอ (Request) ไปถึงการรับข้อมูล (Response) กลับมา รวมถึง บทบาทของ Status Code (เช่น 200, 404, 500) และเหตุใดการใช้ HTTPS จึงสำคัญสำหรับความปลอดภัย

เมื่อผู้ใช้พิมพ์ URL ลงในเบราว์เซอร์ เช่น https://... .com แล้วกด Enter กระบวนการจะเริ่มดังนี้

1. Browser จะส่งคำขอ Request
2. Server รับคำขอ แล้วจะประมวลผลตรวจสอบว่าผู้ใช้ต้องการอะไร (เช่น หน้าเว็บ)
3. Server ส่ง Response กลับมา อาจเป็น HTML, CSS, JavaScript หรือรูปภาพ
4. Browser นำข้อมูลที่ได้มาแสดงเป็นหน้าเว็บให้ผู้ใช้เห็น

#### บทบาทของ Status Code

- 200 OK → สำเร็จ โหลดหน้าเว็บได้ปกติ
- 404 Not Found → ไม่พบหน้าเว็บ
- 500 Internal Server Error → เซิร์ฟเวอร์มีปัญหา
